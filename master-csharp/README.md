## Generics

Generics allow creating classes, methods, and interfaces with **type parameters**.  
They increase **code reusability**, **type safety**, and **performance**.

---

## Delegates and Events

Delegates are **type-safe references to methods**.  
Events are built on delegates and enable the **publish-subscribe pattern**.

---

## LINQ (Language Integrated Query)

LINQ enables querying collections in a **SQL-like syntax** directly in C#.  
It simplifies working with data from **arrays, lists, XML, databases, and more**.

---

## Asynchronous Programming

Async programming allows writing **non-blocking code** for better responsiveness and scalability.  
Key concepts include **async**, **await**, and **Tasks**.

---

## Reflection

Reflection allows inspecting **assemblies, types, and members at runtime**.  
It is useful for **dynamic behavior, plugins, and meta-programming**.

---

## Attributes

Attributes provide **metadata about code** that can be used during runtime or compile-time.  
They are useful for **annotations, validation, serialization, and more**.

---

## Dependency Injection

Dependency Injection (DI) allows **injecting dependencies instead of creating them manually**.  
It improves **modularity, testability, and maintainability**.

---

## Design Patterns

Common patterns used in C# include:  
- Singleton  
- Factory  
- Repository  
- Observer  
- Strategy  

Design patterns help write **scalable, reusable, and maintainable code**.

---

## Advanced Memory Management

- **Garbage Collector (GC):** automatic memory management.  
- **IDisposable / using:** deterministic resource cleanup.  
- **Span<T> and Memory<T>:** high-performance memory handling.
